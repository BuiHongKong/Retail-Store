System deployed on AWS inside a VPC. An Application Load Balancer routes traffic to ECS Fargate services — 1 frontend and 5 backend. Data persisted in RDS PostgreSQL via Prisma ORM. Items marked ⭐ exist in production only (dashed outline).

Two separate GitHub repositories manage deployments. Staging deploys automatically on every push to main. Production requires a manual promote step, plus separate manual workflows for rollback, database seed, and load testing.

Additional manual workflows

All production workflows use workflow_dispatch — no auto-triggers. The promote step is an explicit human gate between staging and production.
